Intro

It's been a long time since I've made a post, and I felt like the best way to convey the changes would be to use some format that allows me to incorporate both words and images.

This may not be a permanent format, but I would like to try to document my changes in a more interesting way, and a blog post seemed like it might be a decent method.

I'm going to try and capture most of the main changes/features, but there will likely be a lot of things left out / that I will bring up in the future - so stay tuned!

Gameplay

Objectives

I've reworked the objective system - which doesn't change any look from the outside, but it makes it a lot more feasible for me to develop future objectives.

Abilities

For abilities, I'm transitioning from a class-based approach to a more generic approach like done in Vampire Survivors.

My thought process for this is that it will allow for more variety in each playthrough, the players will not be limited to using X abilities with Y class, but instead have the option to mix and match different abilities as they see fit.

That being said, there will still be some ability differences, which leads into my next headline of...

Classes

Currently, there are four classes, of which each will have distinct features.

  1. Each class should play around some unique feature or be otherwise distinct
  2. Each class should have a unique way of survival -> in this case regaining health
  3. Each class should have access to unique meta-progression upgrades
  4. Each class should be able to access some kind of survival ability

That being said, for this version nothing really changes about the classes. Builder and Mage currently have no survival mechanisms.

The builder's turrets however are now functioning on a timer, so you cannot simply camp in a corner without having to replace turrets.

Level Design

Maps

For maps, I've purchased three asset packs for three separate environments. I was hesitant to do this at first, but I felt that these assets aligned well with my vision of what I wanted to do for the levels.

I haven't finished the level design yet, and it's something that I struggle with, but here is some visuals to look at for them:

Long-term, the plan for the levels is to have one unique endless level and one unique campaign level for each environment type. The idea for the levels is that the campaign level will have structured objectives to complete - still maintaining the simplicity of a survivor game but objectives such as:

  • Move to a specific location
  • Stay in a specific zone for a while
  • Survive for X seconds
  • Kill X elite enemies
  • Kill the boss

Tutorial

In addition, we now have a quick tutorial to explain active abilities, movement, and picking up items.

The tutorial also features a secret!

Moving on to events, I've added a new system where over time different types of events will occur during the player's survival experience.

These events will eventually be more diverse, but currently we have three unique events:

  1. Swarm: Enemies rapidly spawn near the player using the current enemy pool
  2. Wave: Flying enemies spawn on a directed path, travelling through the player's given position
  3. Circle: A circle of enemies spawn around the player with high HP, trapping them

User Interface

Character and Level Selection

I've made another design change to the character and level selection screens. I've opted to separate the selections for level and character to allow for more space to present each idea separately.
